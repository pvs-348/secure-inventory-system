About This Workflow

This page demonstrates the complete secure query lifecycle based on the Harn Identity-Based Multi-Signature scheme and RSA encryption. When the Procurement Officer(PO) queries the quantity of an inventory item, all four inventory nodes jointly sign the result using Harn multi-signature. The PKG verifies the signature and encrypts the result using the Procurement Officer's public key. The PO then decrypts and independently verifies the response.

Run the steps from top to bottom, each step depends on the previous one. Records must be added via Task 1+2 before querying here.

Query Input

The Procurement Officer enters an Item ID to query. The system will look up that item across all inventory nodes.

1

Submit Query

The Procurement Officer submits the query to the PKG. The PKG forwards it to all inventory nodes which search their local records. A consistency check confirms all nodes agree on the result. The quantity value becomes the message m that will be signed and encrypted. 

Waiting for query submission.
2

PKG and PO Key Derivation

Derives RSA parameters for both the PKG and the Procurement Officer using the hardcoded values from the List of Keys document. Computes n = p × q, phi(n) = (p-1)(q-1), and the private key d = e-1 mod phi(n) for each party. 

Waiting for key derivation.
3

Secret Key Generation

The PKG generates a secret key for each inventory node by signing their identity with the PKG private key. Formula: gi = IDid_pkg mod n_pkg. Each node's identity from the List of Keys (126, 127, 128, 129) acts as their public key in the Harn scheme. The secret key gi is sent privately to each node by the PKG. 

Waiting for secret key generation.
4

T Value Computation

Each inventory node uses its assigned random number ri (from the List of Keys) to compute its commitment value: ti = rie_pkg mod n_pkg. The nodes exchange their ti values and all compute the aggregate: t = tA × tB × tC × tD mod n_pkg. 

Waiting for t value computation.
5

Hash Computation: H(t, m)

Computes the hash H(t, m) used in the Harn signing step. The aggregate t and message m are concatenated as strings, then the base-256 integer encoding (the same method used in Task 1+2) is applied. This binds the signature to both the commitment value and the specific query result. 

Waiting for hash computation.
6

Partial Signature Computation

Each inventory node independently computes its partial signature using its secret key and random number: si = gi × riH(t,m) mod n_pkg. The nodes broadcast their si values and all compute the aggregate: s = sA × sB × sC × sD mod n_pkg. The final multi-signature is the pair {t, s}. 

Waiting for partial signature computation.
7

PKG: Consensus Check

After all inventory nodes compute their partial signatures, the PKG checks that every node arrived at the same aggregate {t, s} values.

This is the consensus step for Task 3. Here all nodes run the same deterministic math with the same inputs so their results must be identical if everyone is honest.

The PKG compares each node's submitted {t, s} side by side. If any node's values differ that node has cheated or made an error and is rejected. If all 4 nodes agree then only the PKG proceed to cryptographic verification. 

Waiting for consensus check.
8

PKG: Multi-Signature Verification

Once consensus passes the PKG cryptographically verifies the multi-signature using the PKG public key. This proves all 4 nodes signed correctly.

LHS: se_pkg mod n_pkg
RHS: (IDA × IDB × IDC × IDD) × tH(t,m) mod n_pkg

If LHS equals RHS, the multi-signature is valid and the PKG proceeds to encrypt the result. 

Waiting for verification.
9

PKG: Encrypt Result for Procurement Officer

The PKG encrypts the approved query result using the Procurement Officer's PUBLIC key. Formula: c = me_po mod n_po. Only PO who holds the matching private key can decrypt this ciphertext. The PKG sends the package { Enc(m, PK_PO), t, s } to the Procurement Officer. 

Waiting for encryption.
10

Procurement Officer: Decrypt and Verify

The Procurement Officer uses their private key to decrypt the ciphertext: m = cd_po mod n_po. The PO then independently verifies the multi-signature by running the same verification equation:

LHS: se_pkg mod n_pkg
RHS: (IDA × IDB × IDC × IDD) × tH(t,m) mod n_pkg

If LHS equals RHS, the PO confirms the result was collectively approved by all 4 inventory nodes. 

Waiting for decryption and verification.
